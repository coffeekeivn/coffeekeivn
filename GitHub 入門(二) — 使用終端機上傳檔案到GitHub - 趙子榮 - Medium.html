GitHub 入門(二) — 使用終端機上傳檔案到GitHub

趙子榮
Mar 25, 2017 · 5 min read

剛好要上傳作業，我就順便寫下了此篇文章…

首先打開你的終端機，你會發現你的目錄在 MacBook-Pro:～Username＄，

我的使用者名稱叫alan2305，所以我的畫面中是MacBook-Pro:～alan2305＄

同時，我下了一個止令’’ ls ’’(是L不是1)，它可以告訴你目前所在的位址有什麼檔案

接下來，第一步：建立一個專案並複製有紅色方框的網址（專案網址）

第二步，選擇你要建立專案的地方

我想在桌面建立專案，所以我下了指令 cd desktop，cd 就是“進入”的意思，順帶一提， cd .. 就是”跳出“所在的目錄或資料夾，由（圖一）可知，alan2305的目錄有桌面這個資料夾，我要進入桌面，我可以：

MacBook-Pro:～alan2305＄ cd desktop

第三步，把專案clone下來

這步驟就是上圖畫紅線的部分，打 git clone + 專案網址

上圖表示clone成功，我的專案 “Twitch”也如下圖出現一個資料夾在桌面上

接著打開資料夾，你會發現有一個README.md，我們可以編輯一下這個檔案（非必要流程）

我把作業的註解寫到此檔案～～

第四步，把你要傳的檔案拖到專案(Twitch)的資料夾下

之後，從桌面(desktop)進入專案資料夾(Twitch)

第六步，使用終端機在你的專案目錄(Twitch)下，打上指令：

git status(縮寫為 git st) 就是確認專案資料夾裡面是否有檔案新增或刪除，

出現很多紅色文字，表示些些東西都是剛剛新增的，GitHub沒有這些資料

第六步，下指令 git add .(add空格加dot)，把剛新增的檔案加入

結果沒反應～不用擔心：）

第七步，再下一次指令 git status，確認剛新增的檔案是否成功加入

一堆綠色字體，表示你成功新增檔案…

第八步，git commit -m “隨便打”

上傳剛剛新增的檔案到 local repository(你的電腦的倉庫)，雙引號中的文字將會成為你每次上傳的紀錄代號，我習慣打 “first upload” ，“second upload” …

看到這些數量嚇人的檔案並沒有紅色的ERR表示commit成功！

第八步，git push

把剛上傳到local repository的資料再傳到remote repository(GitHub的倉庫)

看到100%表示上傳完成！！

最後…我們確認一下

檔案上傳成功～贊贊贊！！

同時…

我們剛剛在README.md檔案加的註解也成功秀在GitHub上，
